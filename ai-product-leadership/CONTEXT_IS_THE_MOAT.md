# Context Is the Moat

## Definition

The principle that AI-driven products become defensible not through model capability, but through the integration of proprietary context and data.

## Core Principle

> Models are interchangeable. Context is not.

## System Structure

1. Model Layer
   - interchangeable
   - rapidly commoditizing

2. Context Layer
   - proprietary data
   - user-specific inputs
   - historical information

3. Integration Layer
   - how context is fed into the model
   - workflow embedding

## Key Insight

Without context, outputs are generic.

With context, outputs become differentiated and valuable.

## Competitive Dynamics

| Factor | Commodity | Differentiator |
|-------|----------|---------------|
| Model | Yes | No |
| Context | No | Yes |
| Workflow Integration | No | Yes |

## Failure Mode

- building AI features without context
- relying solely on model capability
- creating easily replicable features

## Differentiation

Winning products:
- integrate deeply with user data
- generate outputs tied to real workflows
- produce results competitors cannot replicate

## Application

Use this model to:

- design AI product features
- evaluate competitive advantage
- prioritize integration over tooling
